How To´s: ⇒ Back to T.O.C.

In this section i will try to answer some comon questions people could have.
  1. How does the de-centralisation of the information work?
    It works like this:
    By creating a JavaScript file and storing it on your own computers hard-drive then telling the application to process that script file in its execution.
    This way, you are able to write script commands that the application will use without modifying the actual application files.
    The application will process this local file as if it was part of the application, without the need to change any code in any file of the application or the need to upload the local file to the website the application is on.
    Only you are able to see the effects of this localfile because only you have it, its on your computer and is not shared with other people.
  2. How do I create a JavaScript file?
    You can create a JavaScript file using any ASCII editor but when you save the text file give it a file-extension of ".js".
    In JavaScript files you can have comments also.
    Single line comments start with a double-slash // and span until the end of line.
    Multi line comments start with a slash-star /* and span until a star-slash */ is encountered.
    For example:
    Useing M$´s Notepad you can create a textfile and save it as "MyFile.js"
    Notes:
    1. M$´s Wordpad is NOT the same application.
    2. When you dont see file extensions first enable it, or the file gets an automatic ".txt" as extension extra appended !
  3. How do I add a planet?
    You can do this useing a single line like:
    Universe.add( new Planet("Seregor", 6) );
    This adds a planet with Seregor as name and tells the application to display 6 minimaps per line on planet view mode.
  4. How do I add a map to a specific planet?
    You can do this useing a single line like:
    Universe.planet("Seregor").add( new Map("Liyon Valley", 5120,5120, "Seregor_3_4") );
    This will add a map named Liyon Valley to a planet named Seregor.
    It sets the mapsize to be 5120px width and 5120px high.
    And uses map_Seregor_3_4.png as mapimage in detailed view.
    And uses map_Seregor_3_4_small.png as mapimage in planet view.
  5. How do I add a mark to a specific map?
    You can do this useing a single line like:
    Universe.planet("Seregor").map("Liyon Valley").add( new Mark("Map|Portal|Planet", 1181, 544, "WorldPortal", "To Glandatoo:Lilaci City") );
    This will add a mark named WorldPortal to a map named Liyon Valley on a planet named Seregor.
    It sets the marks description to be To Glandatoo:Lilaci City.
    And also sets the marks position to be 1181px horizontal and 544px vertical relative to the lower-left corner of the map.
    Furthermore it sets the mark to be one of the default marktypes Map|Portal|Planet, which automaticaly assigns an image to the mark and sets its zoom levels for display or hide.
    Notes:
    1. For a overview of available selectors browse the application using "MarkTypes" as an argument in the search string of the URL.
      Example: ?MarkTypes or simply click here.
    2. If there are other search string arguments already present just append "&MarkTypes" to the search string of the URL.
      The "&" is used to seperate arguments in search strings...
  6. How do I use the Local-File feature?
    You can use this feature by following these steps.
    1. Use the info in the preceeding How-To questions.
    2. Right click this link and choose "Save Target As" or "Save Link As" depending on your browser.
      And save the file as "C:/Marks.js".
    3. Modify / Edit the file you just saved to your needs.
      It is only a template, to get you started with comments.
      You can/should edit the, commented, text after "Planet:", "Map:" and "Contributed by:".
      This is to let me and others know what the file´s purpose is and who you are, when you finaly decide to submit it to me for inclusion on the website.
    4. Navigate the map page using "LocalFile" as an argument in the search string of the URL.
      Notes:
      • If there are other search string arguments already present just append "&LocalFile" to the search string of the URL.
        The "&" is used to seperate arguments in search strings...
    5. If your browser´s security does not allow local file linking from a online webpage then see the next HOW-To on how to bypass this.
  7. My browser does not allow local file linking from a online webpage, what now?.
    This is someting you should be greatfull about, as it is a security problem under normal circumstances.
    However incase of this application there is nothing to worry about because you are in control what is being executed in this localfile.
    Ok so how do we bypass this security block? Well its rather simple and logical.
    All we need to do is start the application from a local webpage file by following these simple steps.
    1. Right click this link and choose "Save Target As" or "Save Link As" depending on your browser.
    2. Save the file on your local hard drive, anywhere you like.
      I suggest one of two locations for easy finding:
      • Your Desktop
      • The root of your (first) Hard drive. eg: "C:/".
    3. Use the file you just saved to start the application from now on instead of the online URL.
      This will make your browser accept local file linking because the page is stored localy also.
      Although the rest of the application is still fetched online, it does not bother your browsers security.

Application objects explained: ⇒ Back to T.O.C.

There are a couple of methods this applicaton provides for use, which are all defined in ClassLib.js.
  1. MarkTypes object
    This object is the instance of the MarkType object-class and holds all default mark-types.
    It is a non-linear multi-dimensional array kindof type, meaning sub-dimensions can all have individual number of elements that can differ from other dimensions at that level.
    Thus to traverse all elements in all dimensions you need to use recursion and follow each dimension to its end.
    It should not be used standalone except in MarkTypes.js.
    You initialize it by using:
    MarkTypes = new MarkType("MarkTypes");
    This initializes the main MarkTypes object and gives it a name of "MarkTypes".
    Provides:
    Methods:
    • add(sName, sImg, nMinZoom, nMaxZoom)
      Adds a new MarkType object to its subdimension.
      And sets its (Selector)name to the string sName or "UnKnown" if none was supplied.
      And uses sImg as its image. Can be some text or full URL.
      If sImg was not supplied, noMark will be used.
      If sImg was supplied as a non-filename/URL then it will be composed by prepending "MT_" and appending ".png" to sImg.
      Use the img.src variable to fetch the actual filename/URL.
      Thus in case sImg was not supplied the actual filename/URL will be: MT_noMark.png (noMark defaults to "noMark")
      nMinZoomand nMaxZoom Are the Min/Max Zoom level allowed to show mark on map.
    • get(nInd)
      Returns the MarkType object with index# nInd from its immidiate subdemension.
      If nInd was not supplied it returns the last MarkType object added to this object´s immidiate subdemension.

    Public variables:
    • name = String containing the selector name for the marktype or "UnKnown" if none was suplied.
    • length = Total length of sub-types Array that is being hold in Obj.
      This is done for convienience in traversals.
    • minZoom, maxZoom = Min/Max Zoom level allowed to show mark on map. (-1 being minimap zoomlevel.)
    • Obj = A Javascript Array() object holding al subtypes consisting of MarkType object´s.
    • img = A Javascript Image() object holding the image to use representing this marktype.
    ⇒ Back to T.O.C.
  2. Mark object
    This object holds everything about a mark.
    You create one by using one of these lines:
    new Mark(sType, nX, nY)
    new Mark(sType, nX, nY, sName)
    new Mark(sType, nX, nY, sName, sDesc)
    Where:
    • sType = Map Image. Can be some text or full URL/filename.
      If supplied as non-URL/filename then it should be composed of selectors into the main MarkTypes object.
      When using more as one selector then you should seperate them using a vertical dash, '|'. Examples:
      "NPC"
      "NPC|Quest"
      "Map|Building|WaterPut"
      For a overview of available selectors browse the application using "MarkTypes" as an argument in the search string of the URL.
    • nX = Horizontal position in pixels in the map. (0 = left edge)
    • nY = Vertical position in pixels in the map. (0 = bottom edge)
    • sName = Optional but reccomended.
      String with this Mark´s name.
    • sDesc = Optional but reccomended.
      String explaining what this Mark does or is for.
      Can hold HTML tags.
    You can fetch a Mark object by using one of these lines:
    Map.get("markname")
    Map.get(sName)
    Where:
    • Map = A Map object returned by the map() method of the Planet object.
    • sName / "markname" = A string with the name of the mark used to create it.
    Provides:
    Public variables:
    • Map = The Map object this mark belongs to.
      This can be used for reverse traversals.
    • type = The full marktype selector or "Custom" when initialized using a URL/Filename.
    • img = A javascript Image() object holding this marks image.
      This is used so the browser pre-loads the assosiated image of the mark.
    • x, y = X,Y Coordiantes of the mark in the map.
    • name = Name of the mark or "NoName" when none was supplied.
    • desc = String containing the description of the mark or null when none was supplied.
      Can hold HTML tags.
    • minZoom, maxZoom = Min/Max Zoom level allowed to show mark on map. (-1 being minimap zoomlevel.)

  9. Universe object
    This is the instance of the main SolarSystem application object.
    The object´s class name is SolarSystem because thats the name of the application.
    You initialize it by using one of these lines:
    Universe = new SolarSystem( sName, null, sImg );
    Universe = new SolarSystem( sName, aPlanets, sImg, aIMAP );
    Where:
    • sName = SolarSystem´s name.
    • aPlanets = Optional Array of Planet object´s.
    • sImg = SolarSystem Image. Can be some text or full URL.
    • aIMAP = Optional Array of ImageMapArea object´s to use as Imagemap.
    Provides:
    Methods:
    • clear()
      Clears the Array of Planet object´s and ImageMapArea object´s to use.
    • image(bflag)
      Returns full filename to use as image. If the provided sImg was not a full URL this will compose a full filename,
      by prepending "map_" and, if bflag=true, appending "_small" to it and using ".png" as extension.
    • add(obj)
      Adds a Planet object or ImageMapArea object depending on the type of obj provided.
      This can be used to add these objects after initialisation.
    • planet(sName)
      Returns a Planet object corrosponding to sName.
      This is used to fetch Planet object´s from the Universe object.
    • Control = A ControlCenter object used as a interface to interact with DHTML functionalities.

    Public variables:
    • name = Name of the planet.
    • planets = A Javascript Array() object holding all Planet object´s for this SolarSystem.
      Use planets.length to get total number of Planet object´s present.
      or planets[nInd] to get a specific Planet object with index# nInd.
    • img = SolarSystem Image. Can be some text or full URL.
      Use the image() method to get the full URL/filename to use.
    • imgMap = A Javascript Array() object holding all ImageMapArea object´s that makeup the Imagemap for this Universe object´s img.
      Use imgMap.length to get total number of ImageMapArea object´s present.
      or imgMap[nInd] to get a specific ImageMapArea object with index# nInd.
    ⇒ Back to T.O.C.
  10. Planet object
    This object holds everything about a planet.
    You create one by using one of these lines:
    new Planet("planetname")
    new Planet("planetname", x)
    new Planet("planetname", "img")
    new Planet(sName, sArg, aIMAP)
    Where:
    • sName / "planetname" = String with Planet´s name.
    • sArg / x / "img" = Optional either a number or imagename.
      if provided as a number, x, it sets the number of maps to show on one line in planet view.
      if provided as a non-number, "img", it sets the image to use as planet layout. Can be some text or full URL.
    • aIMAP = Optional Array of ImageMapArea object´s to use as Imagemap.
    You can fetch a Planet object by using one of these lines:
    Universe.planet("planetname")
    Universe.planet(sName)
    Where:
    • sName / "planetname" = a string with the name of the planet used to create it.
    Provides:
    Methods:
    • clear()
      Clears the Array of Map object´s to use.
    • image(bflag)
      Returns full filename to use as image. If the provided sArg was not a number or full URL, this will compose a full filename,
      by prepending "map_" and, if bflag=true, appending "_small" to it and using ".png" as extension.
    • add(oMap)
      Adds a Map object to the Array of Map object´s to use.
      This can be used to add these objects after creation.
    • map(sName)
      Returns a Map object corrosponding to sName.
      This is used to fetch Map object´s from the Planet object.
    • makeIMAP(sImg, aIMAP)
      This is used to associate a image and imagemap with the Planet object after creation.
      Where:
      • sImg = String containing the image to use as planet layout. Can be some text or full URL.
        This is used by the image() method see above.
      • aIMAP = Array of ImageMapArea object´s to use as Imagemap.

    Public variables:
    • SolarSystem = The SolarSystem object this planet belongs to.
      This can be used for reverse traversals.
    • name = Name of the planet.
    • img = Planet Image. Can be some text or full URL.
      Use the image() method to get the full URL/filename to use.
    • imgMap = A Javascript Array() object holding all ImageMapArea object´s that makeup the Imagemap for this Planet object´s img.
      Use imgMap.length to get total number of ImageMapArea object´s present.
      or imgMap[nInd] to get a specific ImageMapArea object with index# nInd.
    • x = Max number of maps to show on one line in planet view.
    • maps = A Javascript Array() object holding all the Map object´s of this Planet object.
      Use maps.length to get total number of Map object´s present.
      or maps[nInd] to get a specific Map object with index# nInd.
    ⇒ Back to T.O.C.

      Map Image. Can be some text or full URL.
    You can fetch a Map object by using one of these lines:
    Planet.map("mapname")
    Planet.map(sName)
    Where:
    • Planet = A Planet object returned by the planet() method of the Universe object.
    • sName / "mapname" = a string with the name of the map used to create it.
    Provides:
    Methods:
    • clear()
      Clears the Array of Mark object´s to use.
    • image(bflag)
      Returns full filename to use as image. If the provided sImg was not a full URL this will compose a full filename,
      by prepending "map_" and, if bflag=true, appending "_small" to it and using ".png" as extension.
    • add(oMark)
      Adds a Mark object to the Array of Mark object´s to use.
      This can be used to add these objects after creation.
    • get(sName)
      Returns a Mark object corrosponding to sName.
      This is used to fetch Mark object´s from the Map object.

    Public variables:
    • Planet = The Planet object this map belongs to.
      This can be used for reverse traversals.
    • name = Name of the map.
    • x, y = X,Y Sizes of the map.
    • img = Map Image. Can be some text or full URL.

    • Marks = A Javascript Array() object holding all Mark object´s in this map.
      Use Marks.length to get total number of Mark object´s present.

  12. ImageMapArea object
    This object is similar to the HTML AREA element, but has a small variant on the location attribute.
    You create one by using one of these lines:
    new ImageMapArea("shape", Array(shape-coords), "alt text")
    new ImageMapArea(sShape, aCoords, sAlt, sLoc)
    Where:
    • sShape / "shape" = String containing the shape of the area.
      This is the same as the shape names used in HTML.
    • aCoords / Array(shape-coords) = An Array consisting of numbers only.
      These are the same numbers one would using in the HTML version, corrosponding to the sShape.
    • sAlt / "alt text" = String containing the ALT text to use, and is internaly also copied to the TITLE attribute.
      This text is shown when the browsing user moves the mouse over the image part.
    • sLoc = String containing the location to navigate to.
      It consists of 1 part for planet navigation and 2 parts for map navigation seperated by a vertical dash. Examples:
      "Planet"
      "Planet|Map"
      Where:
      • Planet = Name of the planet to navigate to.
      • Map = Name of the map to navigate to.
    Provides:
    Public variables:
    • Shape = String holding the HTML shape name.
    • Coords = A Javascript Array() object holding all coordinates for the Shape.
    • Alt = String holding text to be used as ALT and TITLE texts.
    • Loc = String containing location to navigate to.
      This can be a planet name followed by an optional mapname seperated by a vertical dash "|".
    ⇒ Back to T.O.C.

Overview of Script Parts: ⇒ Back to T.O.C.

The whole application is devided into subsections, and each subsection loads the next section with some exceptions as noted below.
  1. LoadScripts.js
    This is the first script that gets loaded.
    Its only purpose is to provide the loading mechanism and visual.
    It is put in the head section of the HTML file using a script tag.
    The first time LoadScript() is called it will generate and insert the visual into the document.
    From then on it will check the properties of its supplied structure with an interval of 500ms.
    When the values reach their destination it will exit this interval removing the visual and call the init() function of the scripts that used it.
    Provides:
    function LoadScript(sURL, bDefer)
    Where:
    • sURL = String containing URL of script filename to load.
    • bDefer = Boolean indicating usage of the ´defer´ flag.
    structure gLoadScript
    • ToLoad = Total file number to load.
    • TotalSize = Total Bytes to load.
    • Loaded = Current #files loaded.
    • SizeLoaded = Current #Bytes loaded.
    ⇒ Back to T.O.C.
  2. SolarSystem.js
    This is the first script of the actual application.
    It initializes the total files and bytes to load, then loads "TestBrowser.js" and "ClassLib.js", and exits by increasing the loaded bytes with the filesize of itself.
    The initialized total files is actualy 1 less as the total number of files because this file is not loaded using the loader system.
    It is put in the body section of the HTML file using a script tag.
  3. TestBrowser.js
    This script tests the capabilities and type of the users browser.
    It doesn´t load other scripts and exits by increasing the loaded bytes with the filesize of itself.
    You can/should use ShowException() in exception handling, this function will use an alert() to inform the user of any exceptions if the global boolean bDebug is set true.
    Provides:
    function ShowException(e)
    Where:
    • e = A Javascript Error() object.
    Global variables:
    • bDebug = true/false indicating Debugging functions should be active.
    • noDOM = true/false indicating no support of the W3C DOM model.
    • noDOM1HTML = true/false indicating no support of the W3C DOM-HTML Level 1 model.
    • noDOM2 = true/false indicating no support of the W3C DOM-Core Level 2 model.
    • noDOM2HTML = true/false indicating no support of the W3C DOM-HTML Level 2 model.
    • isIE4, isIE5, isIE6 = true/false indicating use of Internet Explorer browser versions.
    • isNS4 = true/false indicating use of Netscape v4 browsers.
    • isMoz = true/false indicating use of Mozilla browsers.
    • isFF, isFF09, isFF010 = true/false indicating use of FireFox, FireFox v0.9 and FireFox v0.10 browsers respectivly.
    • isOp = true/false indicating use of Opera browsers.
    • isMac, isMacIE = true/false indicating use of Macintosh browsers.
    • gTestBrowser = null or a string with HTML to be included in a document.
      The HTML consists of a notice and results of DOM tests.
    ⇒ Back to T.O.C.
  4. ClassLib.js
    This is the heart of the application.
    It provides all the objects that are used to setup the whole system.
    It loads "Support.js", and exits by increasing the loaded bytes with the filesize of itself.
    Provides:
    Global variables:
    • noMark = Image placeholder to use when no mark image available. (defaults to "noMark")
    • noMap = Image placeholder to use when no map image available. (defaults to "back")
    • MarkTypes = Main object of default MarkTypes.
    • Items = Main object of Items.
    • Monsters = Main object of Monsters.
    • NPCs = Main object of NPCs.

    Global objects:
    • See: Application objects explained
  5. Support.js
    This script consists of all kind of functions that are used in inter-connections of the objects in "ClassLib.js", as well as functions for the test, URL-search and init() functionalities.
    It loads "DHTML.js", and exits by increasing the loaded bytes with the filesize of itself.
    Provides:
    Global variables:
    • Universe = Main application object.
    • bDebug = true/false indicating Debugging functions should be active.
    • bTest = true/false indicating Testing mode active.
    • bCC = true/false indicating if ´Control Center´. should be shown
    • bMarkTypes = true/false indicating if main ´MarkTypes´. listing should be shown
    • gShowMaps = true/false indicating if ´Maps´. should be shown
    • gShowMonsters = true/false indicating if ´Monsters´. should be shown
    • gPlanet = Initial Planet to navigate to.
    • gMap = Initial Map to navigate to.
    • gZoom = Initial Zoom Level to use in detailed view.
    All of these, except Universe, are set using the URL-Search functionality.
    ⇒ Back to T.O.C.
  6. DHTML.js
    This script is the heart of all visual representations.
    It consists of functions that deal with the generation of the DHTML content.
    It loads "MarkTypes.js", "Items.js", "Monsters.js", "NPCs.js" and exits by increasing the loaded bytes with the filesize of itself.
  7. MarkTypes.js
    This script initializes the Main MarkTypes object of default MarkType´s.
    For a overview of created selectors, browse the application using "MarkTypes" as an argument in the search string of the URL.
    It loads "Planets.js", and exits by increasing the loaded bytes with the filesize of itself.
  8. Planets.js
    This script initializes the Main Universe object of the SolarSystem application.
    It populates the Universe object with Planet object´s and populates them by loading their corrosponding "Maps xxxx.js" files.
    It exits by increasing the loaded bytes with the filesize of itself.
